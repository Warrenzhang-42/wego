# WeGO - AI 本地导览产品方案文档

> **愿景**：让每一次旅行都成为一场跨越时空的文化深度对话。

## 1. 产品概述
**WeGO** 是一款定位于"下一代 AI 智能游伴"的移动端旅行应用。它打破了传统旅游 App 仅提供"信息堆砌"的局限，通过大语言模型（LLM）与地理位置服务（LBS）的深度融合，为用户提供私人口语化、具备深厚文化底蕴的本地导览服务。

### 1.1 产品双锚定位
WeGO 的产品价值锚定在两个核心维度的交汇点上，这也是区别于所有竞品的根本差异：

| 维度 | 定义 | 竞品现状 | WeGO 实现 |
| :--- | :--- | :--- | :--- |
| **情绪价值（AI 陪伴）** | 让旅行不再孤单、不再无聊 | 工具化的导航/攻略，无情感连接 | 人格化 AI 导游，口语化讲解，知你所好 |
| **实用价值（LBS 导览）** | 真实解决"去哪、怎么走、看什么" | 信息堆砌，缺乏即时性与深度 | 空间感知讲解，实时路线规划，深度知识库 |

> 当"情绪价值"与"实用价值"在同一个交互界面中无缝融合时，用户获得的不是一个工具，而是一个**有温度的文化旅行搭档**。

### 1.2 产品核心价值 (Core Value)
*   **去攻略化**：无需由于信息过载而产生的复杂预调研，AI 个性化定制路线，随走随看。
*   **深层次体验**：挖掘城市背后的非遗、历史与市井故事，而非仅仅是网红打卡。
*   **沉浸式交互**：陪伴式导游，支持语音/文字实时交互，具备性格色彩的 AI 形象。

## 2. 市场痛点分析
*   **信息同质化**：传统平台推荐高度趋同，难以满足个性化、分众化的审美需求。
*   **导览缺失**：自由行用户在古迹或历史街区往往"看个热闹"，缺乏专业的即时讲解。
*   **过度工具化**：现有地图或旅游工具缺乏情感温度，交互割裂。

## 3. 核心功能模块

### 3.1 智能路线探索 (Smart Route Discovery)
*   **主题化分类**：提供 Citywalk、老字号/中医药探秘、非遗体验、艺术漫游、自然探险等多元标签。
*   **动态热度系统**：基于实时大数据反馈路线受欢迎程度。
*   **微缩地图缩略**：不仅展示封面图，通过 SVG 动态生成路线路径预览，直观展示体力消耗与耗时。

### 3.2 AI 随行导游 (AI Companion)
*   **多性格人格化**：用户可选择不同语言风格的导游（如：地道京腔的老北京、专业的艺术馆馆长、亲切的旅行博主）。
*   **情境感知讲解**：基于 GPS 定位，当用户进入景点 30 米半径范围时，AI 自动推送相关背景知识（地理围栏触发）。
*   **多模态交互**：支持长按录音发送、文字输入，提供包含"知识卡片"、"景点卡片"、"美食推荐"在内的丰富交互内容。
*   **智能意图路由**：Agent 自动判断用户问题类型 —— 深度文化问题查询内部知识库（Local RAG），即时性问题（排队/票价/天气）调用外网搜索。

### 3.3 深度知识图谱 (Knowledge Repository)
*   **非遗活化**：深入挖掘如"兔儿爷"、"剪纸"等传统文化的历史脉络与制作工艺。
*   **传承人档案**：引介当地手工艺人、非遗传承人，为文化传播建立连接。
*   **RAG 驱动**：所有知识以结构化切片存储，AI 讲解时基于用户位置进行空间过滤 + 语义匹配 + 关键词增强的三重混合检索，确保"所见即所听"。

### 3.4 旅程回顾与收获 (Trip Summary)
*   **成就系统**：旅程结束后自动生成"收获卡片"，包含行走的里程、探索的知识点。
*   **打卡地图**：用户的全部打卡记录在地图上以发光勋章样式渲染，形成个人文化足迹。
*   **社交传播**：极简美学的海报生成功能，一键分享至社交平台。

## 4. 技术方案

> 完整的技术选型论证与架构设计详见 [WeGO_Technical_Solution.md](./WeGO_Technical_Solution.md)。以下为核心选型摘要。

### 4.1 前端与地图
*   **前端技术栈**：Vanilla JavaScript, CSS3, Semantic HTML5（Mobile-First）。
*   **地图引擎**：三图全接入（Mapbox / 高德 / 百度），通过统一 Adapter 抽象层屏蔽差异，用户可自主切换。
*   **坐标系统**：后端统一 WGS-84，前端使用 gcoord 库按引擎实时转换（GCJ-02 / BD-09）。
*   **跨平台策略**：Web 先行验证 -> Phase 2 使用 **Capacitor** 零改动打包为 iOS/Android App，支持后台定位与息屏播报。

### 4.2 AI 引擎与知识库
*   **Agent 编排**：采用 **LangGraph**（LangChain）实现多工具编排与状态管理。
*   **RAG 检索增强**：旅游知识按景点/主题切片，通过 Embedding 模型向量化后存入 **Supabase pgvector**，支持语义搜索 + 空间过滤 + 关键词增强的三重混合检索。
*   **LLM**：Claude Sonnet 4 / GPT-4o，支持复杂推理与人格化表达。
*   **AI 路线规划三步分离原则**：
    1.  AI 生成结构化 POI 节点序列（JSON）
    2.  LBS API 校验真实距离与耗时（防止 LLM 空间幻觉）
    3.  前端渲染真实轨迹线（Polyline）

### 4.3 后端与数据库
*   **BaaS 选型**：**Supabase**（PostgreSQL + pgvector + PostGIS + Auth + Realtime + Edge Functions），一库解决关系数据、向量检索与空间计算。
*   **实时推送**：通过 **SSE（Server-Sent Events）** 实现 Agent 讲解的流式推送。
*   **状态管理**：前端通过 `sessionStorage` 暂存对话进度，服务端通过 `agent_transcripts` 表持久化完整交互历史（JSONB 格式存储结构化卡片）。

## 5. 设计美学 (Design Aesthetics)

### 5.1 视觉风格
*   **色彩体系**：
    *   **主色**：故宫红 (#b22314)，彰显文化底蕴与热情。
    *   **辅助色**：优雅的象牙白 (#FFF8E8)、静谧紫 (#F4F0FF)、生命绿 (#EBF7F0)，用于区分不同类别的文化版图。
*   **字体设计**：
    *   **西文**：Manrope (现代、高可读性)。
    *   **中英配比**：Noto Serif SC (宋体)，用于标题，传达书卷气质与历史庄重感。

### 5.2 交互美学
*   **半透明磨砂 (Glassmorphism)**：底部悬浮导航、对话气泡使用适度的毛玻璃效果。
*   **微动效 (Micro-animations)**：热度圆点呼吸效果、分类 Chip 点击涟漪、卡片入场渐隐渐现。

## 6. 项目路线图 (Roadmap)

### Phase 1：Web MVP 核心验证（已完成 + 进行中，4 周）
*   完成首页分类探索与热门路线展示。
*   实现路线详情页（含 mini Map 路径渲染）。
*   搭建 AI 陪伴导游对话界面原型。
*   搭建 Supabase 后端，创建核心数据库表结构。
*   实现 Map Adapter 抽象层（首接高德）。
*   构建第一批知识库数据（杨梅竹斜街路线）。
*   接入 LLM API，实现基础 RAG 对话。

### Phase 2：多引擎 + Agent 智能化（4 -> 8 周）
*   完成 Mapbox / 百度 Adapter 接入，用户可自由切换。
*   实现 Agent 自主路线规划（含约束校验：时长、步行距离）。
*   上线打卡系统与地图标记联动。
*   增加导游人格选择功能。
*   地理围栏主动触发讲解。

### Phase 3：Capacitor 打包 + 后台能力（8 -> 12 周）
*   集成 Capacitor，打包 iOS / Android App。
*   接入后台定位插件（息屏场景）。
*   实现 TTS 语音播报。
*   上架应用商店。

### Phase 4：商业化与生态
*   集成路线订单支付功能。
*   开启非遗传承人线上店铺模块。
*   支持全球多语言路线拓展。
